普通编写spring的方式

普通的获取对象的方式，所有的对象之间的依赖，类之间的依赖关系都是在java代码里面维护的，很难维护的，如果我们有替换方案，替换比较困难，对象的产生全是在配件文件里面的

  • setter注入（最常用的） 必须其字段有对应setter方法才可以完成name setName() ,通过property子节点完成的注入

我们可以导入配置文件处理器，以后编写配置就有提示了

运行application.properties配置文件也是可以的，不过会乱码，要在Settings下找到File Encodings (文件编码)

@ImportResource:导入Spring的配置文件，让配置文件里面的内容生效:
Spring Boot里面没有Spring的配置文件，我们自己编写的配置文件，也不能自动识别；
想让Spring的配置文件生效，加载进来;@ImportResource标注在一个配置类上

@ImportResource(locations = {"classpath:beans.xml"}}
导入Spring的配置文件让其生效

然后再 hexo deploy

问题：hexo配置好后，运行hexo clean && hexo g && hexo d命令，出现ERROR Deployer not found: git

原因：少了hexo针对git的deploy组件

解决方法：npm install –save hexo-deployer-git

搞定，运行成功之后提示：INFO Deploy done: git
