面向对象

发表于 2019-09-02 更新于 2019-09-03
  • 面向对象是相对面向过而言
  • 面向对象和面积过程都是一种思想
  • 面向过程，强调的是功能行为
  • 面向对象：将功能封装进对象，强调具备了功能的对象
  • 面向对象是基于面向过程的

对象：将功能放到了其中 ，不用找其它的单个功能，因为功能已收进对象中

三个特征：封装，继承，多态

开发就是找对象用，没有对象，就创建一个对象

类和对象的关系

类就是对现实生活中事物的描述，对象就是这类事物实实在在的实体

类：

映射到java中，描述的是class定义的类

对象：

对象对应java在堆内存中用new建立实体储存

属性对应的是类中的变量，行为对应的是类中的方法

其实定义类，就是在描述事物，就是定义属性和行为，属性和行为成为类中成员变量和成员方法

成员变量和局部变量

作用范围：

  1. 成员变量作用于整个类中
  2. 局部变量作用于函数中，或语句中

在内存中的位置

成员变量：在堆内存中，因为对象的存在，才在内存中存在，与类同在

局部变量：在栈内存中

匿名对象

封装

指隐藏对象的属性各实现细节，仅对外提供公共访问方式

好处：

  • 将变化隔离
  • 便于使用
  • 提高重用性
  • 提高安全性

封装原则：

  • 将不需要对外提供的内容都隐藏起来
  • 把属性都隐藏，提供公共方法对其访问

每当有一个私有的属性，都有set和get

构造函数

特点：

  1. 函数名与类名相同
  2. 不用定义返回值类型
  3. 不可以定return语句

作用：
给对象进行初始化

  • 默认构造函数的特点
  • 多个构造函数是以重载的形式存在的

构造函数是与类名一致的，一建立就会自动调用

构造函数的作用，可以用于给对象进行初始化

当自定义了构造函数，系统就不给分配了

构造函数和一般函数在写法上有不同

在运行上也有不同

构造函数在对象一建立就运行，给对象初始化的一般方法是对象调用才执行，是给对象添加具备的功能

一个对象建立，构造函数只运行一次

而一般方法可以被对象调用

构造代码块

作用：给对象进行初始化

对象一建立就运行，而且优于构造函数执行

和构造函数的区别：

构造代码块是给所有对象进行统一初始化

构造函数是给对应对象初始化

this关键字

this代表本类的对象

this代表它所在函数所属对象的引用

哪个对象在调用this所在的方法，this就代表哪个对象

this是用于区分局部变量和成员变量同名情况

但凡本类功能内部使用了本类对象，都用this

this只能放在构造方法的第一行，因为初始化要先执行，this只能在构造方法间

Static

static是一个修饰符，用于修饰成员(成员变量，成员函数)，

  • 使堆内存中重复的变量，变为公有的，

  • 除了可以被对象调用外，还可以直接被类名调用，形式是：

    类名.静态成员。

static特点：

  1. 随着类的加载而加载
  2. 优先于对象的存在
  3. 被所有对象所共享
  4. 可以直接被类名调用

实例变量和类变量的区别 ：

  1. 存放位置：

    • 类变量随着类的加载而存在于方法区中，随着类的消失而消失
    • 实例变量随着对象的建立而存在于堆内存中

  2. 生命周期 ：

    • 类变量生命周期最长，随着类的消失而消失
    • 实例变量生命周期随着对象的消失而消失

静态的使用注意事项：

  1. 静态方法只能访问静态成员

    非静态方法既可以访问静态也可以访问非静态,因为静态是先出来的，

    它后面的可以访问它

  2. 静态方法中不可以定义this，super关键字

    因为静态优先于对象存在，所以静态方法中不可以出现this

静态有利有弊

利：

  • 对对象的共享数据进行单独空间的存储，节省空间，没必要一个对象存储一份
  • 可以直接被类名调用

弊：

  • 生命周期过长，访问出现局限性，只能访问静态
  • 主函数是静态的
